# Repository Guidelines

## プロジェクト構成とモジュール配置

このリポジトリは、Three.js で作られた最小構成の静的な 360 度パノラマビューアです。

- `index.html` はページ本体、UI コントロール、import map、モジュールの入口を定義します。
- `main.js` はビューア初期化、テクスチャ読み込み、OrbitControls 設定、画像アップロード、ズーム、リセット、全画面、リサイズ処理を担当します。
- `style.css` はレイアウトと UI スタイルをまとめます。
- `assets/panorama.jpeg` は初期表示用の equirectangular パノラマ画像です。可能な限り 2:1 比率の画像を使ってください。

現時点ではビルドシステムやパッケージ管理設定はありません。新しいワークフローで明確に必要な場合を除き、追加ツールは最小限にしてください。

## ビルド・テスト・開発コマンド

- `python3 -m http.server 8000` はリポジトリルートでローカル静的サーバーを起動します。
- `http://localhost:8000` をブラウザで開くとビューアを実行できます。

通常の確認では `index.html` をファイルシステムから直接開かないでください。ブラウザ制限により JavaScript モジュールや画像読み込みが失敗する場合があります。

## コーディングスタイルと命名規則

- HTML、CSS、JavaScript は 2 スペースインデントを使います。
- JavaScript は ES Modules と `const` / `let` を優先し、現在の単純なモジュール状態を超えるグローバル変数は避けます。
- 変数名と関数名は `loadPanorama`、`resetView`、`currentTexture` のような説明的な camelCase にします。
- CSS セレクタは `#viewer`、`#ui`、`.upload`、`.hint` など既存の ID / class に沿ってシンプルに保ちます。
- UI テキストは、意図的なローカライズでない限り日本語を維持してください。

## テスト方針

自動テストフレームワークは未設定です。変更後はローカルサーバーを起動し、ブラウザで手動確認してください。

最低限、次を確認します。

- `assets/panorama.jpeg` から初期パノラマが読み込まれる。
- ドラッグで視点が回転する。
- マウスホイールでページスクロールせずにズームできる。
- 画像アップロードで一般的な画像を選択でき、2:1 でない場合に警告される。
- リセットボタンと全画面ボタンが動作する。
- デスクトップ幅と狭いモバイル幅の両方でレイアウトが使える。

## コミットとプルリクエストの方針

このリポジトリにはまだコミット履歴がないため、固有のコミット規約は推定できません。`Add upload validation` や `Fix mobile control layout` のように、簡潔な命令形のメッセージを使ってください。

プルリクエストには次を含めてください。

- ユーザーに見える変更の短い要約。
- 手動テスト手順と使用したブラウザ。
- UI 変更がある場合はスクリーンショットまたは録画。
- 新しいアセット、外部依存、CDN バージョン変更がある場合の補足。

## セキュリティと設定の注意

シークレットやローカル環境ファイルはコミットしないでください。将来設定を追加する場合は `.env.example` にプレースホルダー値を記載し、実際の `.env*` ファイルは非公開にしてください。
